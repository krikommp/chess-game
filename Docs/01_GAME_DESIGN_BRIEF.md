# 01 - 游戏设计纲要 (Game Design Brief)

> 本文件定义**核心玩法骨架**，是所有其他设计文档的上位约束。修改本文件需要明确确认。

## 1. 类型与视角

- **类型**：3D 战术战棋（tactics / SRPG）。
- **视角**：45° 俯视（top-down isometric-like）。
- **相机操作**：
  - ✅ 默认跟随当前主控角色
  - ✅ 跟随时允许位置缓动
  - ✅ 拉近 / 拉远（zoom）
  - ✅ 平移（pan）：允许 `WASD / 方向键` 或拖拽移动相机
  - ❌ 旋转（暂不支持；相机朝向固定，不做旋转缓动）
- **聚焦规则**：
  - 角色开始行动时，相机会重新聚焦到角色
  - 若玩家在此后继续手动移动相机，则停止强制聚焦，直到下一次角色行动
- **后续扩展**：旋转可能在后期开放，但当前所有 UI / 关卡设计**按"视角不会旋转"做假设**。

## 2. 战斗结构

### 2.1 无格子移动

- 参考 **博德之门 3 / 神界原罪 2**：无网格，直接点击地面移动。
- 移动距离由 **角色速度值 × 行动点（AP）** 决定（详见 `02_SYSTEM_SPEC.md`）。
- 路径需要寻路系统（NavMesh 候选，待 §06 决定）。

### 2.2 战斗进入方式

- **不切换场景**。在当前探索地图上原地进入战斗。
- 触发方式（待补，列入 OPEN_QUESTIONS）：可能是接近敌人 / 主动攻击 / 剧情触发。

### 2.3 回合制

- **行动顺序**：按 **先攻值（initiative）** 从大到小依次行动。
- **一轮 (round)**：所有参战单位都行动一次结束。
- **轮次结束后**：重新按先攻值排序进入下一轮。
- **先攻值稳定性**：进入战斗时确定，整场战斗内**不会动态变化**（除非未来设计明确允许，例如 buff/debuff，待定）。

### 2.4 行动点（AP）

- 每个单位每回合获得若干 AP（具体上限待定，写在 `02_SYSTEM_SPEC.md`）。
- **移动消耗 AP**：根据角色速度值换算（1 AP 能走 X 米）。
- **攻击消耗 AP**：基础攻击的消耗待定。
- **技能消耗 AP**：每个技能自己定义消耗。
- 一回合内 AP 用完 / 主动结束 → 进入下一单位。

## 3. 玩家阵营

- 玩家可操控一个或多个角色；设计上不再设置固定 4 人上限。
- 原型阶段的快捷键、UI 或场景配置可以先覆盖有限数量的角色，但这只是交互实现限制，不是队伍规模规则。

## 4. 失败 / 胜利条件

- 待定。先按"全队倒下 = 失败 / 全部敌人击败 = 胜利"做最简假设，写入 `02_SYSTEM_SPEC.md`。

## 5. 不在本期范围内的（明确排除）

- 多人联机
- 实时战斗 / ATB
- 自由旋转相机
- 切换战斗场景
- 格子化棋盘移动
